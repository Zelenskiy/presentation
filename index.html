What is a browser extension API

A browser extension API is a set of programming interfaces provided by web browsers for developing extensions to modify and enhance browser functionality.

Definition of API

An API, or Application Programming Interface, is a set of protocols and tools that allows software applications to communicate with each other by defining methods and data structures for interaction.

Role of browser extension APIs

Browser extension APIs enable developers to extend the functionality of web browsers by providing access to browser features and resources, allowing for the creation of custom functionalities.

Types of extensions:

Chrome, Firefox, Edge, etc.

How browser extensions work

Principle of extension operation

Browser extensions operate by injecting custom scripts into web pages, interacting with the browser through APIs to modify page content and behavior. They can also run background scripts independently of the active web page

Extension life cycle: installation, usage, removal

The extension life cycle consists of installation, where users add the extension to their browser, followed by usage, during which the extension enhances the browsing experience. Eventually, users may choose to remove the extension from their browser.

Key features of extension APIs

Manipulating web pages

Extension APIs offer methods for dynamically manipulating web pages, enabling developers to customize content, styles, and interactions. This involves injecting scripts, modifying DOM elements, and enhancing the functionality of web applications.

User interaction

Extension APIs facilitate user interaction by enabling developers to create intuitive interfaces with UI components such as buttons and menus. This allows users to engage with extension features through inputs, events, and feedback mechanisms, enhancing usability.

Network operations

Extension APIs support network operations, enabling developers to send HTTP requests and interact with remote servers. This functionality allows extensions to fetch data, interact with APIs, and integrate external services within the browser environment.

Local storage

Extension APIs offer local storage mechanisms for persisting data within the browser. This allows developers to store user preferences and application state, ensuring a seamless browsing experience across sessions.

Popular uses of extension APIs

Ad blocking and tracking protection

Ad blocking and tracking protection extensions are widely used applications of extension APIs. They utilize browser functionality to detect and block ads, pop-ups, and tracking scripts, enhancing user privacy and browsing experience.

Password managers

Password managers are popular extensions that utilize extension APIs to enhance security and convenience for users. They securely store and autofill passwords, generate strong passwords, and offer additional features like syncing and multi-factor authentication support.

Productivity enhancements

Productivity enhancement extensions utilize extension APIs to streamline tasks and improve efficiency for users. They offer a range of tools from to-do lists to advanced project management features, leveraging browser functionality to integrate with web applications and automate workflows.

Website functionality extensions

Website functionality extensions enhance specific websites or web applications by adding new features or modifying existing ones. They leverage extension APIs to interact with page content, integrate with third-party services, and provide custom functionality tailored to users' needs.

Usage examples

How to Write Your Own Basic Browser Extension

Now let's start by writing a very basic extension.

To begin, we'll create a folder inside which we create a file named manifest.json

What is a manifest file?

A manifest file is required for any browser extension. This file contains basic data about our extension, such as name, version, etc.

Now copy the following fragment into the manifest.json file:

              
                  {
                    "manifest_version":2,
                    "version":"1.0",
                    "name":"Test",
                  }

How to load the extension file

For Chrome users:

In the address bar search for this: chrome://extensions

Enable Developer Mode and switch into it

Click the Load unpacked button and select the extension directory

You've installed the extension successfully. But the extension doesn't do anything currently. Now let's add some functionality to our extension.

To do this, we'll edit our manifest.json file like this:

              
                {
                  "manifest_version":2,
                  "version":"1.0",
                  "name":"Test",
                  "content_scripts":[
                    {
                     "matches":["<all_urls>"],
                     "js":["main.js"]
                    }
                  ]
                }

In the above code, we added a content script to manifest.json. Content scripts can manipulate the Document Object Model of the web page. We can inject JS (and CSS) into a web page using a content script.

"matches" contains the list of domains and subdomains where the content script should be added and js is an array of the JS files to be loaded.

Now inside the same directory create a main.js file and add the following code: 

              
                alert("The test extension is up and running")

Now reload the extension and when you visit any URLs you will see an alert message.

How to Customize Your Browser Extension

Now let's have some more fun with our extension.

What we are going to do now is create a web extension that changes all the images of a webpage we visit to an image we choose.

For this, just add any image to the current directory and change the main.js file to: 

              
                console.log("The extension is up and running");
  
                var images = document.getElementsByTagName('img')
  
                for (elt of images) {
                  elt.src = `${browser.runtime.getURL("pp.jpg")}`
                  elt.alt = 'an alt text'
                }

Where can it be used? Interesting question. Maybe for people with special needs

I almost forgot. You still need to make changes to the file manifest.json here 

                
                  {
                    "manifest_version":2,
                    "version":"1.0",
                    "name":"Test",
                    "content_scripts":[
                    {
                    "matches":["<all_urls>"],
                    "js":["main.js"]
                    }
                    ],
                    "web_accessible_resources": [
                      "pp.jpg"
                    ]
                  }

Now just reload the extension and go to any URL. You should now see all images change to the image that is in your current working directory.

How to add icons to your extension

Add the following code in the manifest.json file:

              
              "icons": {
                "48": "icon-48.png",
                "96": "icon-96.png"
              }

How to add a toolbar button to your extension

Now we'll add a button located in the toolbar of your browser. Users can interact with the extension using this button.

To add a toolbar button, add the following lines to the manifest.json file: 

            
              "background":{
                "scripts":["background.js"]
              },
              "permissions":[
                "tabs"
              ]

Then we'll create a new file named background.js in the current working directory and add the following lines in the file: 

            
              function openTab(){
      
                var newTab = browser.tabs.create({
                  url:'https://github.com/Zelenskiy',
                  active:true
                })
              }
              
              browser.browserAction.onClicked.addListener(openTab)

Now reload the extension!

Whenever someone clicks the button, it calls the openTab function which opens a new tab with the URL that links to my github. Also, the active key, when set to true, makes the newly created tab the current tab.

Note that you can use APIs provided by browsers in the background script.

Recommendations for extension development

Using browser documentation

Browser documentation is a crucial resource for extension developers. It offers detailed information about available APIs, methods, and usage examples, helping developers understand how to integrate browser features into their extensions effectively.

Extension project structure

The extension project structure is essential for organizing code, assets, and resources effectively. It typically includes directories for scripts, stylesheets, images, and configuration files. A well-defined structure aids in code maintenance, collaboration, and ensures compatibility across browsers.

Debugging and testing

Debugging and testing are essential for ensuring the quality and reliability of an extension. Developers can use browser developer tools and logging techniques for debugging, while testing involves manual interaction and automated testing to validate functionality, performance, and compatibility across various environments.
